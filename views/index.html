Project Description

The Wildlife Sighting Tracker is a web application built with Express.js that allows users to manage and explore wildlife observation data. This application serves as a digital logbook for wildlife enthusiasts, researchers, and conservationists to record and analyze animal sightings.

Key Features:

  • View all wildlife sightings in a structured format
  • Filter verified vs unverified sightings
  • Search for specific species (like eagles)
  • Filter sightings by habitat (forest, wetlands, etc.)
  • Get statistics and counts for different categories
  • View recent sightings with timestamps

Built with: Node.js, Express.js, HTML5, CSS3, JavaScript (ES6+), Vercel for deployment

API Endpoints

The following RESTful API endpoints are available for accessing wildlife sighting data:

GET /api/sightings

Returns all wildlife sighting records in the database.

GET /api/sightings/verified

Filters and returns only verified sightings (where verified is true).

GET /api/sightings/species-list

Extracts and returns an array of unique species names.

GET /api/sightings/habitat/forest

Returns all sightings from FOREST habitat with count statistics.

GET /api/sightings/search/eagle

Finds the first sighting where species name contains "eagle" (case-insensitive).

GET /api/sightings/find-index/moose

Finds the index position of the sighting with species "Moose".

GET /api/sightings/recent

Returns the 3 most recent sightings with formatted date information.
